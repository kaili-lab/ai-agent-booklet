# 项目协作规则

  ## 知识讲解规范
  - 每个知识点必须包含"知识定位"部分
  - 说明它在上层知识体系中的位置
  - 列出相关概念和前后依赖

  ## 代码注释规范
  - 文件顶部说明在整个体系中的位置
  - 核心代码块说明"为什么这样做"
  - 对比其他实现方式（如果存在）

  ## 文档编写规范（High-Level 优先）
  - **前言（20%）**：说明核心问题、解决方案、方案关系、阅读指引（像书籍前言，让读者看完就知道全貌）
  - **正文（70%）**：直奔主题，只讲"是什么"、"为什么"、"核心权衡"、"适用场景"，不陷入实现细节
  - **总结（10%）**：提炼本质、给出选择指南（对比表格/决策树）、指明下一步
  - **原则**：能用 1 句话说清楚不用 3 句，能用表格不用段落，强调权衡而非完美，假设读者会用 AI 查细节
  - **LangChain 类介绍**：每个功能模块说明使用的 LangChain 类/工具，包括类名、作用、关键方法（简洁介绍，不展开 API 细节）
